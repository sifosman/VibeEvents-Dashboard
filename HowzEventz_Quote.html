4) Technical Approach

  • Mobile App – React Native + Expo (TypeScript) using Expo modules for notifications, media and device APIs.
  • Backend – Supabase: Postgres with Row‑Level Security (RLS), Supabase Auth, and Supabase Storage for media.
  • Server logic – Supabase Edge Functions for secure server actions, webhooks, and integrations.
  • Roles & access – Customer, vendor and admin roles with RLS policies so vendors only edit their own data.
  • Tier entitlements – Store plan info (Free/Premium) in user/vendor metadata; enforce quotas and visible sections per tier.
  • Security – HTTPS everywhere, hardened auth, image checks, geo/rate limiting and moderation hooks.
  • Payments – Selected provider’s secure web checkout (TBD: PayFast/PayGate/Peach/Yoco/Ozow). Merchant of record is the client; funds go to the main company account. Vendor payouts are recorded/managed manually by admin; payout automation is out of scope for MVP.
  • Observability – Sentry for crashes, GA4 for funnels, server logs with alerts, plus Supabase logs/metrics.
  • WhatsApp Deep Link – Deep link to WhatsApp chats (wa.me) with proper Android/iOS handling. Client supplies the support number.
  • POPIA/GDPR – Consent at sign‑up/checkout and store privacy labels.
  • Data retention – Dormant profiles (≥ 3 years) are deleted; favourites kept until cleared; search history kept ~3 months.
  • Reviews workflow – Admin approval queue; support stars + comments + photos.
  • Abuse reporting – In‑app report path; admin triage.
If we need deeper platform features later, we’ll add small native modules (Android/iOS) to the RN + Expo app without derailing the roadmap.
